# 07 · データ処理契約 (DPA) ドラフト

> **目的:** 導入企業の情シス/法務が **ベンダー交渉なしで** そのまま決裁に回せる DPA（Data Processing Agreement）テンプレート。本書をコピーし、社印箇所に押印すれば実効的な合意書として機能します。

⚠️ 注意: 本テンプレートは "speedrun" 用の最小 DPA です。大規模案件 / GDPR 対応が必要な場合は、本書を雛形としつつ法務部門と個別合意書を締結してください。

---

## データ処理契約書

**甲（委託者）:** _____________________（以下「甲」）
**乙（受託者）:** 合同会社 evidai（LemonCake 運営、以下「乙」）

甲と乙は、甲が乙の提供する「**LemonCake for Dify プラグイン および LemonCake API サービス**」（以下「本サービス」）を利用するにあたり、以下のとおりデータ処理契約（以下「本契約」）を締結する。

### 第1条（本契約の目的）

本契約は、甲が本サービスを利用することに伴い、甲から乙に提供される可能性のある個人情報および機密情報の取扱いについて、個人情報の保護に関する法律（以下「**個情法**」）およびその他関連法令を遵守し、適切に処理することを目的とする。

### 第2条（取扱対象情報）

本契約が対象とするのは、甲が本サービスを利用する過程で乙のシステム（`api.lemoncake.xyz`）に送信される以下の情報に限る。

| カテゴリ | 具体例 |
|---|---|
| 認証情報 | Buyer JWT（乙が発行し甲が保管） |
| Pay Token 発行パラメータ | `serviceId`, `limitUsdc`, `expiresInSeconds`, `sandbox` |
| トークン識別子 | `tokenId` |
| 課金問合せパラメータ | `limit`（取得件数のみ） |

なお、本サービスは **甲の Dify ワークフロー内の会話コンテンツ・添付ファイル・ユーザー メッセージ等を送信しない**。この前提が覆った場合、本契約は無効となり、乙は直ちに甲に書面で通知する。

### 第3条（処理目的）

乙は、第 2 条の情報を以下の目的のためにのみ処理する。

1. 本サービスの提供（Pay Token の発行、残高照会、無効化、課金履歴照会）
2. 監査・法令遵守（電子帳簿保存法に基づく 7 年保存等）
3. 重大インシデント発生時の調査および甲への通報
4. 統計分析（個別 Buyer を特定不能な形に匿名化した集計値のみ）

### 第4条（安全管理措置）

乙は、個情法ガイドライン「通則編」に基づき、以下の安全管理措置を講ずる。

1. **組織的**: データへのアクセスを最小権限で管理し、`audit_logs` に全操作を記録
2. **人的**: 従業員との秘密保持契約締結、年 1 回のセキュリティ教育
3. **物理的**: Railway Managed Infrastructure（SOC 2 Type II 取得済みクラウド基盤）
4. **技術的**: TLS 1.2+ / AES-256 / Ed25519 署名 / 冪等性 / 原子的 revoke
5. **外部環境**: 委託先の Railway 社との間でも同等の取扱いを書面で担保

### 第5条（再委託）

乙は、本サービスの提供に必要な範囲で以下の第三者に処理を再委託する。甲はこれに事前同意するものとする。

| 再委託先 | 所在地 | 処理内容 |
|---|---|---|
| Railway Inc. | 米国 | インフラ（計算・ストレージ・ログ） |
| Polygon Labs | 分散 | USDC/JPYC 決済（公開台帳） |
| freee株式会社 他会計ベンダー | 日本/米国 | 甲が明示的に接続した場合のみ、仕訳データの同期 |

乙は、再委託先を追加・変更する場合、**30 日前までに書面で甲に通知**し、甲が異議を申し立てた場合は合理的な代替手段を提案する。

### 第6条（越境移転）

乙は、個情法第 28 条に従い、以下のいずれかの措置を講ずる。

1. 日本国内リージョン (ap-northeast-1) を優先利用
2. 障害時に米国リージョンへフェイルオーバーする場合は、**標準契約条項 (SCC) 準拠の再委託契約**を締結済み
3. 越境移転が発生した場合、甲の求めに応じて具体的内容を開示

### 第7条（保管期間と削除）

1. 乙は、第 2 条の情報を **本契約終了後 30 日以内に削除** する（電子帳簿保存法上の課金レコードは 7 年間保存する義務があるため除く）。
2. 削除の完了を書面または電子メールで甲に通知する。
3. 課金レコードは匿名化処理を施した上で保管する。

### 第8条（事故時対応）

1. 乙は、第 2 条の情報に関する漏えい・滅失・毀損等の事故（以下「**重大事故**」）を認知した場合、以下の SLA で対応する。
   - **初動連絡: 認知から 24 時間以内**
   - **事実関係の調査報告: 72 時間以内**
   - **ポストモーテム公開: 96 時間以内**
2. 乙は、甲が個情法委員会への報告義務を負う場合に備え、必要な情報を速やかに提供する。

### 第9条（監査権）

甲は、本契約の履行状況を確認するため、乙に対し以下を請求できる。

1. **書面監査**: 本監査キット（`docs/dify/audit-kit/`）の更新版の提供（無償・年 1 回まで）
2. **第三者監査**: SOC 2 / ISMS 取得後、証明書のコピー提供（無償）
3. **現地監査**: 事前合意の下、費用は甲負担で実施可能

### 第10条（契約終了）

1. 本契約は、甲が本サービスの利用を終了するまで有効とする。
2. 甲はいつでも 30 日前の通知により本契約を終了できる。
3. 乙は、契約終了後も第 4 条および第 7 条の義務を、該当情報が削除されるまで継続する。

### 第11条（損害賠償）

乙の故意または重大な過失により甲に損害が発生した場合、乙は当該損害を賠償する。ただし、賠償額の上限は **本サービスの過去 12 ヶ月の利用料金相当額** とする（エンタープライズ プランでは個別に上限を合意可能）。

### 第12条（準拠法・管轄）

本契約は **日本法** に準拠し、本契約に関する一切の紛争は **東京地方裁判所** を第一審の専属的合意管轄裁判所とする。

## 付則：用語定義

| 用語 | 定義 |
|---|---|
| 個情法 | 個人情報の保護に関する法律（令和 2 年改正版） |
| SLA | Service Level Agreement |
| SCC | Standard Contractual Clauses（EU GDPR 準拠の標準契約条項） |
| 重大事故 | 本契約第 8 条に定める漏えい・滅失・毀損等 |
| ポストモーテム | 事故後の原因分析と再発防止策を文書化したもの |

## 本 DPA の使い方（情シス向けメモ）

1. 本ファイルをコピーして甲欄に自社情報を記入
2. 法務レビュー（通常 1 週間以内）
3. 乙欄に押印した正本を `contact@aievid.com` 宛に PDF 送付 → 乙の押印を受領
4. 両社で正本 1 部ずつ保管

代替として、**エンタープライズ プランの MSA（Master Service Agreement）** でも同等の保護を提供します。MSA は個別条件調整可能です。
